# Novaqy MVP - Product Requirements Document

## Project Overview
Build a minimalistic, blazing-fast website for Novaqy Cloud LLP, a technical support company serving seniors (55+) across US and Canada. Focus on one-time plan purchases with immediate support call initiation.

## Business Context
- **Company**: Novaqy Cloud LLP (branded as "Novaqy")
- **Target Audience**: Seniors aged 55+ who need patient technical support
- **Geographic Coverage**: United States and Canada
- **Business Model**: One-time plan purchases (not subscriptions) with immediate support fulfillment
- **Support**: Phone +1 800-330-1234 | Hours: 7:30 AM - 7:30 PM PST

## Core Requirements

### Performance Requirements (McMaster-Carr Speed)
- **Page Load Time**: Under 1 second for all pages
- **Lighthouse Performance**: 95+ on all pages
- **First Contentful Paint**: Under 0.5 seconds
- **Bundle Size**: CSS under 50KB, JavaScript under 20KB
- **Static Generation**: All pages pre-built at build time
- **Edge Caching**: Aggressive caching via Cloudflare

### Senior-First Accessibility (WCAG 2.1 AA)
- **Font Size**: 18px minimum throughout platform
- **Contrast Ratio**: 4.5:1 minimum for all text
- **Touch Targets**: 44px minimum for all interactive elements
- **Navigation**: Simple, consistent, maximum 3 levels deep
- **No Auto-play**: No auto-advancing content or videos
- **Clear Language**: Jargon-free, patient explanations
- **Error Handling**: Friendly, actionable error messages
- **Keyboard Navigation**: Full keyboard accessibility
- **Screen Reader**: Complete NVDA, JAWS, VoiceOver compatibility

### Technology Stack
- **Frontend**: Astro 4+ with TypeScript (strict mode only)
- **Styling**: Tailwind CSS 3+ with senior-friendly configuration
- **Components**: Shadcn/ui (curated selection, senior-adapted)
- **Backend**: Supabase (database, auth, real-time features)
- **Hosting**: Cloudflare Pages with edge functions
- **Repository**: GitHub with automated deployments
- **CI/CD**: Jenkins pipeline (typecheck, lint, accessibility, performance)
- **Analytics**: Google Analytics 4
- **Email**: Google Workspace integration

## Supabase Integration

### Database Schema
```sql
-- Contact form submissions
CREATE TABLE contact_submissions (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    name TEXT NOT NULL,
    email TEXT NOT NULL,
    phone TEXT,
    preferred_contact_time TEXT,
    message TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Plan purchases (one-time payments)
CREATE TABLE plan_purchases (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    stripe_session_id TEXT UNIQUE NOT NULL,
    customer_name TEXT NOT NULL,
    customer_email TEXT NOT NULL,
    customer_phone TEXT,
    plan_type TEXT NOT NULL, -- 'mobile', 'individual', 'family', 'family_plus'
    amount_paid INTEGER NOT NULL, -- in cents
    payment_status TEXT DEFAULT 'pending',
    call_initiated BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Call notifications for support team
CREATE TABLE call_notifications (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    purchase_id UUID REFERENCES plan_purchases(id),
    customer_name TEXT NOT NULL,
    customer_phone TEXT NOT NULL,
    plan_type TEXT NOT NULL,
    notification_sent BOOLEAN DEFAULT FALSE,
    call_completed BOOLEAN DEFAULT FALSE,
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Row Level Security (RLS)
- Enable RLS on all tables
- Allow anonymous inserts for contact_submissions
- Restrict plan_purchases to authenticated sessions only
- Admin-only access for call_notifications
